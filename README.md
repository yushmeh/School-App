**НЕКСУС АКАДЕМИЯ** — современная веб-система управления онлайн-школой, построенная на базе **FastAPI (Python 3.14)** с использованием архитектурного паттерна **Clean Architecture (3-Tier)**.

Интерфейс выполнен в виде интерактивного **Single Page Application (SPA)** в футуристичном стиле **Sci-Fi Dark Mode** с эффектами **Glassmorphism**.

## 🚀 Ключевые особенности

### 🏗 Архитектура
- **Clean Architecture / 3-Tier:** Строгое разделение на слои данных, бизнес-логики и представления.
- **JSON-персистентность:** Данные сохраняются в `school_data.json` при каждом изменении. Путь к файлу определяется относительно `repository.py` — данные не теряются вне зависимости от рабочей директории запуска.
- **Автосид при первом запуске:** Если база данных пуста, система автоматически заполняется тестовыми данными — 6 преподавателей, 7 студентов, 6 курсов с вложенными модулями, уроками и домашними заданиями.

### 📚 Модульная структура курсов
Курсы организованы в четырёхуровневую иерархию:
```
Курс → Модуль → Урок → Домашнее задание
```
Каждый уровень создаётся через интерфейс или API и хранится вложено внутри курса.

### 📋 Система домашних заданий
- Преподаватель **выдаёт ДЗ** одному студенту или сразу всем зачисленным на курс.
- Преподаватель **выставляет оценку** (2–5) за каждое выполненное ДЗ с возможностью добавить комментарий.
- **Итоговый балл** по курсу рассчитывается динамически как среднее арифметическое всех оценённых ДЗ.
- **Правило аттестации:** если у студента оценено менее 3 ДЗ на момент завершения курса — в историю записывается **«Н/А»** (Не аттестован).

### 👩‍🏫 Управление преподавателями
- Статусы **«Активен»** / **«В отпуске»**: преподавателя в отпуске нельзя назначить на курс.
- **Защита от удаления:** преподавателя, ведущего активный курс, невозможно удалить из системы.

### 📊 Аналитика и отчёты
- **Отчёт по студенту** — активные курсы, прогресс по ДЗ, история завершённых курсов с финальными баллами и ASCII-визуализацией `[████████░░] 80%`.
- **Отчёт по преподавателю** — активные и завершённые курсы с датами, средняя успеваемость студентов по всем оценённым ДЗ.
- **Сводная аналитика** — общая статистика школы, топ-3 популярных курса, средний балл по школе.

### 📥 Экспорт в CSV
Любой отчёт выгружается одним кликом (с BOM, корректно открывается в Excel):
- Отчёт по студенту — курсы, прогресс, все оценки за ДЗ
- Отчёт по преподавателю — метрики, активные и завершённые курсы
- Отчёт по курсу — структура модулей и уроков, успеваемость студентов

### ✅ Валидация
- Проверка email через regex на уровне Pydantic и в интерфейсе (подсветка ошибки inline).
- Российская шкала оценок **2–5** с выпадающим списком подписей («5 — Отлично» и т.д.).
- Фильтрация курсов по статусу: все / активные / завершённые.

## 📂 Структура проекта

```text
school_app/
├── main.py             # FastAPI app, все API-эндпоинты
├── models.py           # Pydantic-модели и DTO (Course, Module, Lesson, Homework, ...)
├── repository.py       # JSON-персистентность, менеджер состояния БД
├── services.py         # Бизнес-логика, аналитика, CSV-экспорт
├── seed.py             # Заполнение БД тестовыми данными (не затрагивает существующие записи)
├── requirements.txt    # Зависимости проекта
├── pytest.ini          # Конфигурация pytest
├── school_data.json    # Локальная БД (создаётся автоматически)
├── static/
│   └── index.html      # SPA-интерфейс (Sci-Fi Dark Mode / Glassmorphism)
└── tests/
    └── test_main.py    # 52 юнит-теста для всех API-эндпоинтов
```

## 🗂 Схема данных

```
Student               Teacher               Course
├── id                ├── id                ├── id
├── name              ├── name              ├── title / topic
├── email             ├── specialization    ├── status (active | completed)
└── course_history[]  └── status            ├── start_date / end_date
                      (active | vacation)   ├── teacher_id
                                            ├── enrolled_student_ids[]
HomeworkSubmission                          └── modules[]
├── id                                          └── Module
├── homework_id                                     ├── id / title / order
├── lesson_id                                       └── lessons[]
├── module_id                                           └── Lesson
├── course_id                                               ├── id / title / order
├── student_id                                              └── homeworks[]
├── grade (2–5 | null)                                          └── Homework
├── graded_by                                                       ├── id / title
├── comment                                                         └── max_grade
└── assigned_at / graded_at
```

## 🛠 Инструкция по запуску

Для работы необходим **Python 3.14**.

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Запустить сервер из папки school_app/
cd school_app
uvicorn main:app --reload --port 8000

# 3. Открыть в браузере
# http://localhost:8000
```

> При первом запуске с пустой БД система автоматически заполняется тестовыми данными через `seed.py`.

## 📡 API Эндпоинты

### Студенты
| Метод | URL | Описание |
| :---: | :--- | :--- |
| `GET` | `/api/students` | Список всех студентов |
| `POST` | `/api/students` | Создать студента |
| `PATCH` | `/api/students/{id}` | Обновить данные студента |
| `DELETE` | `/api/students/{id}` | Удалить студента |
| `GET` | `/api/students/{id}/card` | Карточка студента с активными курсами и историей |
| `GET` | `/api/students/{id}/stats` | Статистика с ASCII-визуализацией успеваемости |
| `GET` | `/api/students/{id}/submissions` | Все сдачи ДЗ студента (опц. фильтр `?course_id=`) |

### Преподаватели
| Метод | URL | Описание |
| :---: | :--- | :--- |
| `GET` | `/api/teachers` | Список всех преподавателей |
| `POST` | `/api/teachers` | Создать преподавателя |
| `PATCH` | `/api/teachers/{id}` | Обновить данные / сменить статус |
| `DELETE` | `/api/teachers/{id}` | Удалить (запрещено при наличии активного курса) |
| `GET` | `/api/teachers/{id}/report` | Аналитический отчёт по преподавателю |

### Курсы
| Метод | URL | Описание |
| :---: | :--- | :--- |
| `GET` | `/api/courses` | Список всех курсов с деталями |
| `POST` | `/api/courses` | Создать курс |
| `PATCH` | `/api/courses/{id}` | Обновить курс |
| `GET` | `/api/courses/{id}/card` | Подробная карточка курса |
| `POST` | `/api/courses/{id}/complete` | Завершить курс и зафиксировать итоговые оценки |
| `POST` | `/api/courses/{id}/modules` | Добавить модуль в курс |
| `POST` | `/api/courses/{id}/modules/{mid}/lessons` | Добавить урок в модуль |
| `POST` | `/api/courses/{id}/modules/{mid}/lessons/{lid}/homeworks` | Добавить ДЗ в урок |
| `GET` | `/api/courses/{id}/submissions` | Все сдачи ДЗ по курсу |

### Действия
| Метод | URL | Описание |
| :---: | :--- | :--- |
| `POST` | `/api/enroll` | Зачислить студента на курс |
| `POST` | `/api/assign-teacher` | Назначить преподавателя на курс |
| `POST` | `/api/homework/assign` | Выдать ДЗ студенту или всем зачисленным |
| `POST` | `/api/homework/grade` | Выставить оценку за ДЗ |

### Аналитика и экспорт
| Метод | URL | Описание |
| :---: | :--- | :--- |
| `GET` | `/api/analytics` | Сводная аналитика по школе |
| `GET` | `/api/export/student/{id}` | Экспорт отчёта по студенту в CSV |
| `GET` | `/api/export/teacher/{id}` | Экспорт отчёта по преподавателю в CSV |
| `GET` | `/api/export/course/{id}` | Экспорт отчёта по курсу в CSV |

### Служебные
| Метод | URL | Описание |
| :---: | :--- | :--- |
| `POST` | `/api/seed` | Сбросить БД и заполнить тестовыми данными из `seed.py` |
